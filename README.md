# Twitch Auto-Lurker

> Automatically watch, lurk, and chat-greet your favorite Twitch streamers when they go live.

[![License: Apache 2.0](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](LICENSE)
[![Python 3.11+](https://img.shields.io/badge/python-3.11+-blue.svg)](https://www.python.org/downloads/)
[![Platform: Windows](https://img.shields.io/badge/platform-Windows-lightgrey.svg)]()
[![LLM: Ollama](https://img.shields.io/badge/LLM-Ollama-black.svg)](https://ollama.com)

A Python automation tool that polls Twitch for streamers on your watchlist,
opens their stream in your browser when they go live, sends a personalized
lurk message generated by a local LLM, and closes the tab once they've been
offline for a configurable grace period.

Designed to be unobtrusive: tabs open in the background (no focus-stealing
mid-game), messages come from a local Llama model so they don't repeat
across streams or feel bot-like, and the watcher uses your own Twitch
account via OAuth — no separate bot account required.

## Features

- **Live detection** via Twitch's Helix API — polls once per minute by default
- **Background tab opening** through the Chrome DevTools Protocol — no focus steal
- **LLM-generated lurk messages** via a local [Ollama](https://ollama.com) instance
- **Template fallback** from `messages.txt` when the LLM is unavailable
- **Auto-close tabs** after a configurable offline grace period (5 min default)
- **Brief-disconnect handling** — recovered streams don't trigger duplicate lurks
- **OAuth via your account** — messages are sent as you, not from a bot
- **Hot-reload watchlist** — edit `streamers.txt` without restarting

## How it works

```
                 ┌─────────────────────┐
                 │     watcher.py      │  poll every 60s
                 │     (poll loop)     │ ──────────────────────▶  Twitch Helix /streams
                 └─────────┬───────────┘
                           │
        offline → live     │     live → offline
       ┌───────────────────┴───────────────────┐
       ▼                                       ▼
┌─────────────┐                          ┌────────────────────────┐
│ Open tab    │                          │ Start grace timer      │
│ (CDP/bg)    │                          │ (default 5 min)        │
└──────┬──────┘                          └──────────┬─────────────┘
       │                                            │
       ▼                                            ▼ if expired
┌─────────────┐                          ┌────────────────────────┐
│ Ollama LLM  │ → message → IRC chat     │ Close tab (CDP)        │
└─────────────┘                          └────────────────────────┘
```

1. The watcher polls Twitch's Helix `/streams` endpoint every 60 seconds.
2. On **offline → live** transitions:
   - Query CDP for already-open tabs to avoid duplicates.
   - Open the stream in a new background tab (via CDP) if it isn't already open.
   - Generate a lurk message via Ollama, or pick a `messages.txt` template if the LLM isn't available.
   - Send the message over Twitch IRC after a small random delay so the timing doesn't look robotic.
3. On **live → offline** transitions:
   - Start a grace timer (default 5 minutes).
   - If the streamer comes back live within the window, no action — brief disconnects are normal.
   - If the grace expires, close the auto-opened tab via CDP.

## Installation

See [`setup.md`](setup.md) for the complete walkthrough: Python venv, Twitch
developer-app registration, `.env` configuration, Ollama install, browser
debug mode, OAuth flow, and optional autostart.

Quick version:

```powershell
# 1. Create venv and install dependencies
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt

# 2. Register a Twitch app at https://dev.twitch.tv/console/apps
#    Set the OAuth Redirect URL to http://localhost:3000/auth/twitch/callback

# 3. Create .env with your Twitch credentials and chosen model
#    (See setup.md for the exact contents.)

# 4. Install Ollama and pull a model
ollama pull llama3.2:3b

# 5. (Recommended) Bake the debug flag into your Opera shortcut once
.\enable_opera_debug.bat

# 6. First run -- pops a browser for Twitch OAuth
.\start_watcher.bat
```

## Usage

### Daily workflow

After setup:

1. Open your Chromium browser (Opera GX recommended — Chrome/Edge/Brave work too).
2. Double-click `start_watcher.bat`.

Or run `enable_autostart.bat` once and the watcher launches automatically at
every Windows login. `disable_autostart.bat` reverses it.

### Editing the watchlist

`streamers.txt` accepts one URL or bare username per line. Lines starting
with `#` are comments. The watcher re-reads this file each poll, so edits
take effect within a minute — no restart needed.

```
# Watchlist
https://www.twitch.tv/example_streamer
another_streamer
yet_another_username
```

### Customizing fallback messages

`messages.txt` holds the lurk templates used when the LLM is unavailable.
One message per line, `#` comments allowed:

```
# Fallback lurk messages
lurking, have a great stream!
in the background while afk, will catch up later
dropping in to lurk for a bit
```

When Ollama is reachable, these are bypassed in favor of generated messages.

## Configuration

Tunable constants live in [`config.py`](config.py):

| Constant | Default | Purpose |
|---|---|---|
| `POLL_INTERVAL_SECONDS` | `60` | How often to query Twitch for live status. |
| `LURK_DELAY_MIN_SECONDS` / `LURK_DELAY_MAX_SECONDS` | `5` / `15` | Random delay before sending a chat message — avoids robotic 0-second timing. |
| `OFFLINE_CLOSE_GRACE_SECONDS` | `300` | How long a streamer must stay offline before the tab is auto-closed. Set to `0` for immediate close. |
| `SCOPES` | `["chat:read", "chat:edit"]` | Twitch OAuth scopes requested. Changing requires re-authorization. |
| `AI_MODEL` | from `.env` (`AI_BACKEND`) | Ollama model name. |
| `OLLAMA_URL` | `http://localhost:11434` | Where to reach the Ollama daemon. |

## Standalone tests

Each major module has a `__main__` self-test for verifying setup:

```powershell
.venv\Scripts\python.exe twitch_auth.py    # confirm/refresh OAuth tokens
.venv\Scripts\python.exe llm.py            # generate one sample message
.venv\Scripts\python.exe irc_chat.py       # send one message to YOUR own channel
```

`irc_chat.py` defaults to the authenticated user's own channel as a safe
target — you can verify chat sending without spamming someone else's chat.

### Log labels

| Label | Meaning |
|---|---|
| `[STARTUP]` | Info shown once at watcher start. |
| `[LIVE]` | A tracked streamer was already live when the watcher started. |
| `[GO-LIVE]` | A streamer just transitioned offline → live. |
| `[LLM]` | LLM generation attempt (success or fallback). |
| `[LURK]` | Chat message scheduled or sent. |
| `[OFF]` | A streamer just transitioned live → offline (grace timer started). |
| `[BACK]` | A streamer recovered within the grace window — no re-lurk. |
| `[CLOSE]` | Grace timer expired and the tab was closed. |
| `[ERROR]` | A non-fatal error — the watcher continues. |

## Troubleshooting

**`Stored token is missing required scopes …`** — normal after a scope
upgrade. The watcher pops the Twitch authorize page automatically; click
Authorize and it continues.

**`Browser remote debugging not detected on :9222`** — the browser isn't
running with the debug flag. Run `launch_opera.bat`, or run
`enable_opera_debug.bat` once to make the flag persistent. Without CDP the
watcher still works, but tabs open in the foreground and there's no
auto-close.

**`Ollama not reachable`** — the Ollama tray app isn't running. Open it
from the Start menu. The watcher falls back to `messages.txt` templates,
so this is non-fatal.

**Twitch IRC auth rejected** — the token doesn't have the `chat:edit`
scope. Delete `tokens.json` and re-run; the new auth flow will request
the right scopes.

**Brief disconnects causing duplicate lurks** — increase
`OFFLINE_CLOSE_GRACE_SECONDS` in `config.py`. The grace timer suppresses a
second lurk when a streamer recovers within the window.

For setup-time issues, see the dedicated troubleshooting section in
[`setup.md`](setup.md).

## Privacy & security

- `--remote-debugging-port=9222` exposes browser tabs and pages to anything
  on localhost. Acceptable on a personal machine; **do not enable on a
  shared computer**.
- `tokens.json` contains live Twitch access + refresh tokens. Gitignored by
  default. Treat the file like a password.
- `.env` contains the application's Client Secret. Same — gitignored.
- IRC traffic is TLS (port 6697), but chat message *content* is visible to
  anyone in the channel, as with any chat client.
